执行摘要

本周最值得进入 FreeCAD 动力学自动化验证链的变化有三条。第一，FreeCAD Assembly 的仿真视频导出在主分支完成 PyAV 路径重构与 10 帧 MP4 自动测试，并新增 CMake 运行时依赖探测；这使“装配求解 - 可视化 - 证据导出”更可测试，但合并晚于 7 月 15 日周构建，尚不能视为稳定版本能力。

第二，FreeCAD 新增 FEM Python 类型桩，明确暴露 frdToVTK、readResult、isVtkCompatible 等接口，有利于静态检查和 AI/脚本编排。第三，VTK 9.7.0-rc2 开发线集中修复 polyhedron 面信息保留、SurfaceNets 32/64 位输出与 EnSight 科学计数法解析，适合作为 CalculiX/VTK 后处理回归重点。

AI 侧没有足以直接替代求解器的“通用突破”，但出现六项可迁移信号：HierCAD 将生成无效率降至 1.45%；SUFLECA 在 ScanNet25k 上以 33.4%/42.3% 类别/实例准确率领先零样本基线 10.3/12.2 个百分点；DRIFT 报告分布式 FNO 前向 38-64 倍、训练 37 倍加速；层压铁芯 RNN 代理把含磁滞/涡流 FE 成本压到无磁滞仿真的约 2 倍；HyPRNN 的分级多尺度设计将峰值应力降低 42%；agentic neural-operator 搜索在 Navier-Stokes 上以相当精度减少 2.5 倍参数，但论文明确不声称 LLM 带来准确率优势。结论是：优先把 AI 放在候选生成、参数标定和可审计代理层，保留 FreeCAD/CalculiX/VTK 的确定性复算门禁。

本周最优先的 3 项行动

  1. 建立 Assembly 视频导出冒烟测试：在隔离的 FreeCAD weekly/主分支构建中安装 PyAV、Pillow，导出 10 帧 MP4，核对帧数、尺寸、文件关闭和缺依赖时的可诊断错误；不要直接替换生产版本。
  2. 冻结 FEM-VTK 兼容性探针：用新增类型桩约束脚本，并在目标 FreeCAD 包内记录 Fem.getVtkVersion()、Fem.isVtkCompatible()；以含 polyhedron 的 FRD/VTK 样例做转换、单元数、面连通与场数组哈希回归。
  3. 启动“AI 候选 + 确定性复算”小实验：先选 50 个参数化零件/装配任务测试 HierCAD 类序列生成，所有结果必须经过建模命令解析、重算、拓扑有效性、质量属性和约束残差验证；禁止只看渲染图判定成功。

FreeCAD Assembly 自动化与 API/插件变化

官方 weekly 预发布，不是稳定版

【官方预发布】Development Build weekly-2026.07.15 于 7 月 15 日发布，页面明确标记 prerelease=true 并警告不要用于生产。它提供 Windows installer/7z、Linux AppImage、macOS DMG 与 SHA256；因此适合隔离回归，不应被写入“正式发布”基线。

Assembly 仿真视频导出可测试性提升

【已合并开发变更】PR #31345 于 7 月 19 日合并：create_video 明确使用 PyAV，容器采用上下文管理，MP4 优先 libx264 并回退 mpeg4；新增测试生成 10 张 50×50 图像、编码 2 fps MP4，再解码核对 10 帧和尺寸。运行时依赖探测提交还让 CMake 对 PyAV/Pillow、IfcOpenShell 发出缺失警告并报告版本。

工作流影响：视频可成为仿真证据包的一部分，依赖也更早暴露。局限：测试需要 GUI、PyAV、Pillow；只覆盖 MP4，不覆盖 WebM/AVI、透明背景、长序列、失败中断和 Windows 编解码器差异。风险/置信度：中等风险、高证据置信度；代码和测试已合并，但未进入 7 月 15 日 weekly。建议验证：用 10/300/3000 帧三档，检查帧数、首末帧像素哈希、分辨率、耗时、峰值内存与临时文件清理。

FEM 类型桩和受控渲染 API

【已合并开发变更】FEM 类型桩提交明确列出 open/insert/export/read/show、frdToVTK、readResult/writeResult、VTK 版本和兼容性接口；受控图像渲染提交新增 C++ renderToImage(options)，含尺寸、采样、背景、渲染意图及失败返回。

工作流影响：类型桩提升脚本、IDE 与 agent 的可发现性；渲染 API 明确截图契约。局限：类型桩不保证运行时兼容；renderToImage 当前是 C++ API，不能推断已有 Python 绑定或无 GUI 渲染。建议验证：用 mypy/pyright 检查 FEM 脚本，并做 FRD 转换与 GUI 线程探针。

AI × CAD/CAE

HierCAD：层级 Text-to-CAD 与参数落地

【预印本】HierCAD 以 Qwen 2.5-Coder-14B + LoRA 将构造树分成对象级程序推理和零件级拓扑推理，再用结构对齐与参数扰动排序减少数值槽位“抄近路”。数据为 DeepCAD，并采用 Text2CAD/CADmium 注释。结果：Line/Arc/Circle/Extrusion F1 为 93.12/79.43/93.91/99.10；无效率 1.45%，低于 CADmium 的 4.12%；Chamfer mean 35.22，优于 44.52。代码仓库公开。

迁移价值：可把自然语言变为 FreeCAD 参数化命令候选。局限/失效模式：训练用简化 JSON 和单体构造序列，不等于 FreeCAD Assembly 约束图；论文观察到跨字段与字段内参数复用混淆；4 张 RTX A6000 训练约 80 小时。风险/置信度：高部署风险、中高研究置信度。建议验证：50 个由 FreeCAD 重算的盲测任务，统计解析率、重算成功率、无效拓扑率、体积/质心误差、约束残差和人工修复步数。

SUFLECA：CAD-图像几何对齐

【预印本】SUFLECA 用 Normalized Object Coordinates 弱监督和几何一致的一对一匹配，在 12 个真实/合成数据集共 674K 图像上训练。ScanNet25k 的类别/实例准确率为 33.4%/42.3%，比最强零样本基线高 10.3/12.2 个百分点，并报告单对象亚秒级推理；代码仓库公开。

迁移价值：适用于从现场照片/渲染图识别 CAD 实例与初始位姿，辅助装配初始化、几何理解和异常定位。局限：准确率仍不足以自动签署装配；9D 对齐会受遮挡、相似零件、尺度歧义和仿真到真实域偏移影响，也不是 B-Rep 修复算法。风险/置信度：高自动执行风险、中等迁移置信度。建议验证：自建 20 类机械零件、每类 20 个视角与遮挡等级，测位姿/尺度误差，并用 FreeCAD 干涉与配合面残差做二次验证。

Agentic neural-operator discovery：可审计但不等于更准

【预印本】Agentic AI Scientific Community 用 planner/worker/reviewer 虚拟实验室在分段回归、1D advection/Burgers、2D Navier-Stokes/Darcy 五个问题上各重复三次。9,623 次 LLM 调用全部记录，99.8% 规划选择混合架构；与规则基线相比，筛选准确率在 1 个标准差内不可区分，作者明确不声称准确率提升，但 Navier-Stokes 上以相当准确率得到平均 2.2M 参数、相对规则方案 5.6M 少约 2.5 倍。代码、配置和完整转录公开。

迁移价值：适合在固定算力与误差门限内搜索代理架构，并保留审计轨迹。局限：仅五个标准 PDE，搜索成本含大量 LLM 调用；跨任务没有通用赢家，Darcy 与 Navier-Stokes 的最佳结构互不通用。风险/置信度：中等研究置信度、高工程治理要求。建议验证：锁定一个 FreeCAD/CalculiX 梁或转子数据集，给 agent 统一训练预算，和 Optuna/规则搜索比较误差、参数量、GPU 时、复现方差和失败回退率。

DRIFT：分布式 FNO 通信规避

【预印本】DRIFT 以局部部分 DFT + AllReduce/AllGather 只传保留模态，替代先全量分布式 FFT 再截断。3D+time FNO 在 4-32 GPU、最多 8 节点上，相对分布式 FNO 基线报告前向 38-64 倍、训练 37 倍加速，通信占比从 97% 降至 6% 以下，且频谱系数与基线一致。

迁移价值：对高分辨率时空代理和 solver acceleration 有意义。局限：依赖多节点 GPU、GPU-aware MPI 与规则谱网格；论文未给出可直接核验的代码链接，不能把基础设施级加速外推到普通工作站或非结构网格。风险/置信度：中等算法置信度、低近期可迁移性。建议验证：先在 1/2/4 GPU 上复现相同网格和模态数，分别记录通信、算子误差、训练收敛与显存；若 4 GPU 无优势则 No-Go。

层压铁芯 RNN：磁滞/涡流材料代理嵌入 FE

【预印本】RNN laminated-core surrogate 以 GRU 逐时步预测磁场关系，训练数据为 100,000 条人工场序列，验证/测试各 10,000 条；数据生成在单 CPU 核约四天。模型嵌入二维磁矢势 FE，在两个现实应用中将含磁滞与涡流的计算成本控制为无磁滞仿真的约 2 倍，并提供独立公开组件。

迁移价值：这是“可嵌入增量求解器”的参数标定/材料代理，而非离线图片预测。局限：特定 M235-35A 非取向钢、二维各向同性层压模型；历史依赖、极端饱和、训练外波形和自适应步长会造成失效。风险/置信度：中高迁移价值、中等材料泛化置信度。建议验证：用 CalculiX/GetDP 参考环路做训练内/训练外频率、DC 偏置、饱和与变步长盲测，门限同时约束 B-H 环面积、力/损耗和 Newton 收敛。

HyPRNN：物理嵌入多尺度代理与优化

【预印本】HyPRNN 用 hypernetwork 根据椭圆木片长宽比、体积分数、基体剪切模量等微结构变量生成 physics-recurrent network 权重，和完整 FE² 复算对照；分级圆盘优化相对随机微结构降低峰值应力 42%。代码和部分预训练模型公开。

迁移价值：可作为自动参数化、材料分级与拓扑/多尺度优化的候选生成器。局限：二维随机椭圆微结构与特定材料本构，42% 是相对随机基线而非通用拓扑优化优势；数据中不收敛路径会截断。风险/置信度：中等研究置信度、中高域外风险。建议验证：把代理输出回灌 FE²，对最优、次优和域外参数点复算，检查峰值应力、能量、稳定性和制造约束；若排序相关性不足则只作筛选器。

CATIA 与 Rhino / STEP / 3DM 互操作

【本周无重要变化】在窗口内，McNeel rhino3dm 与 OCCT 官方仓库未出现新 commit 或 release；也未发现可由 Dassault 官方材料或可复现工程案例支持的 CATIA-STEP/3DM 新变化。因而本期不重复旧版导入导出能力。

工作流建议：继续使用现有 STEP/3DM 基线样件验证层级、单位、颜色、装配变换、NURBS/B-Rep 有效性和往返质量属性。产品声明若没有具体版本、样件与误差数据，只列入待核验池。

Gmsh 与 meshio

【开发变更】Gmsh 官方下载页仍显示稳定版 4.15.2（2026-03-24）；本周公开镜像提交 Save options: Clipping, Rotation center 扩充选项保存，更偏 GUI/会话可重复性，不是网格算法升级。canonical GitLab 受反机器人挑战保护，因此以官方稳定页和同步镜像交叉核验，置信度中等。meshio 无窗口内提交或 release。

工作流影响：若自动化依赖 Gmsh 视图剪裁或旋转中心，可增加保存/重开回归；网格生成与 meshio 格式桥接无需本周升级。建议验证：保存含 clipping plane/rotation center 的工程，重开后比较选项和截图；同时保留 .msh 2.2/4.1 的节点、单元、physical group 回归。

VTK 与 ParaView

【开发候选】VTK 在 7 月 16 日提交中将版本更新为 9.7.0.rc2，但官方 releases API 在窗口内没有正式 release，故不能写成稳定发布。和 CAE 直接相关的修复包括：OverlappingCellsDetector 保留 polyhedron faces、非结构网格序列化 polyhedron faces、SurfaceNets3D 生成 32/64 位 cells/stencils、EnSight 科学计数法解析修复。

ParaView 本周修复退出时空成员崩溃，合并Python 自动补全模块，并用 vtk::runtime 处理 C++20 编译。影响是后处理稳定性和脚本体验改善；风险是候选线 ABI/数据语义仍可能变化。建议验证：用含 polyhedron、EnSight 指数、FRD 转 VTK 的固定样例，在现有版与候选版比较 cell/face 数、数组范围、截图和 Python 过滤器输出。

CalculiX 与 Abaqus-compatible 流程

【开发变更】CalculiX 作者仓库本周有三项连续提交：自感计算推进、磁能计算重构并新增 induction3 参考结果、RECUR/ELME 文档更新。正式网站仍显示 2.23，所以这些属于开发线，不是稳定发布。

工作流影响：电磁-结构耦合和线圈/电感案例值得关注；Abaqus-compatible 输入兼容不能从单个 induction3 测试外推。局限：提交信息简短、尚无 release notes，参考结果变化需要物理量与单位审查。风险/置信度：中等风险、中高代码存在性置信度、物理正确性待验证。建议验证：固定 induction3 输入，在 2.23 与开发版比较磁能、自感、FRD 字段、能量守恒和网格收敛；再用 FreeCAD 导出的 .inp 做关键字兼容回归。

Python / C++ / pybind11 / ABI

本周的主要 ABI 信号不是 pybind11 本体，而是宿主打包链。FreeCAD 的运行时 Python 依赖探测会在配置阶段实际导入模块并报告版本；VTK Python wrapper 缺失改为 CMake warning，降低“静默缺功能”的概率。VTK wheel CI 将 Python 3.10-3.13 测试迁到 manylinux_2_28，意味着 Linux glibc 基线和 wheel 验证环境需要显式记录。pybind11 唯一窗口内提交只是文档依赖升级，不构成 ABI 变化。

建议：把运行清单最少固定为 FreeCAD commit/tag、Python minor、Qt、OCCT、VTK、PySide、PyAV、Pillow、编译器、C++ 标准、glibc/MSVC runtime；启动时执行 import + 版本 + 最小对象往返测试。不要把“能导入”当成“ABI/数据语义兼容”。

来源与证据边界
